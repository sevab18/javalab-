# javalab-

## Lab 1: Основы ООП

- Созданы базовые классы `Product` и `Category`.
- Реализована основная логика для управления товарами и категориями.

## Lab 2: Инкапсуляция и Валидация

- **Инкапсуляция:** Все поля в классе `Product` сделаны приватными (`private`).
- **Геттеры:** Для всех полей добавлены публичные геттеры.
- **Защищенные сеттеры (Guarded Mutators):**
  - Вместо обычных сеттеров реализованы методы `trySet...()`, которые возвращают `boolean` (`true` при успехе, `false` при ошибке).
  - Добавлена валидация для всех полей `Product` (ID, имя, цена, количество и т.д.).
- **Обновлены методы инвентаризации:**
  - `addStock()`, `sellProduct()`, `applyDiscount()` теперь используют защищенные сеттеры и возвращают `boolean`.
- **Вычисляемый статус запасов:**
  - `getStockStatus()` теперь является вычисляемым полем на основе количества товара.
- **Безопасность в `Category`:**
  - Метод `addProduct()` в классе `Category` теперь возвращает `boolean` и предотвращает добавление `null` или дубликатов.
- **Демонстрация:** Файл `ShopDemo.java` обновлен для демонстрации как успешных, так и неудачных операций, подтверждая работу валидации.

## Lab 3: Конструкторы и Статические Фабрики

- **Перегруженные конструкторы:**
  - Реализованы три конструктора для класса `Product`: без аргументов, с обязательными аргументами (`id`, `name`, `price`) и полный конструктор.
  - Использована цепочка конструкторов (`this(...)`) для избежания дублирования кода.
  - Внутри конструкторов используются методы `trySet...()` для валидации входных параметров, сохраняя значения по умолчанию при невалидном вводе.
- **Статические члены:**
  - Добавлена константа `DEFAULT_CURRENCY` ("KZT").
  - Добавлен статический счетчик `SEQ` для автоматической генерации ID продуктов (`AUTO-1`, `AUTO-2` и т.д.).
  - Добавлен статический счетчик `createdCount` для отслеживания количества созданных объектов `Product`.
  - Реализован метод `getCreatedCount()` для получения общего количества созданных продуктов.
- **Статические фабричные методы:**
  - `Product.of(String id, String name, double price)`: Эквивалентен конструктору с обязательными аргументами.
  - `Product.freeSample(String name)`: Создает продукт с автогенерированным ID, указанным именем, ценой 0.0 и количеством 1.
- **Демонстрация:** Файл `ShopDemo.java` обновлен для демонстрации работы всех новых конструкторов, статических фабричных методов, обработки невалидных параметров в конструкторах и вывода общего количества созданных продуктов.
